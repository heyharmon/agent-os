# Experiment 002 (lean cut): run record + TAKEAWAY

Date 2026-06-16. Per `experiments/PROCESS.md` and
`experiments/002-capable-personal-assistant/charter.md`. This is the lean
first cut: its primary job is to validate the anti-overfit + tournament
machinery cheaply while taking a real step up the spectrum.

Config and full numbers: `results/scorecard-002-lean.md` (all costs read from
provider JSON envelopes). Doer/checker/judge all `claude-sonnet-4-6`. TRIALS=2
per task, hermetic scratch brain per trial. Two worlds: dev (Marisol Vega,
D1-D6) and held-out (Julian Reyes, H1-H5, authored blind, run only at
conclusion).

---

## What ran

A divergent two-architecture tournament on one fixed benchmark:

- **A1, single agent + code-gate** (carried from 001): one agent, file brain,
  plain-text retrieval, the checked harness gate (escalation-produces-artifact,
  writes-via-CLI).
- **A2, doer + checker** (two-agent, divergent bet): one agent does the work,
  a separate checker agent reviews actions against the contract and bounces a
  correction if the escalation artifact is missing/off-topic or a write
  bypassed the CLI.

Each architecture ran the full dev taskset (iterated against), then the
held-out taskset (run only once, at conclusion). 24 trials total per arch path
(see scorecard).

Both architectures pass the held-out bar (target ~80%, 2 trials) with a 0-point

## TAKEAWAY

### Primary result: the machinery validated end to end.

This was the lean cut's main job, and it worked. Every anti-overfit and
tournament mechanism the charter named ran and produced a coherent,
provider-JSON-backed result:

- **Dev/held-out split** held: the system was tuned only on dev; held-out was
  run once at conclusion.
- **Two worlds** (Marisol Vega dev, Julian Reyes held-out) from one template,
  different personas and data.
- **Blind held-out authoring**: H1-H5 were written by an agent told only the
  use case and persona shape, not the architectures or scoring.
- **Divergent-architecture tournament**: A1 (code-gate) vs A2 (doer+checker)
  on the same fixed benchmark.
- **Generalization-gap reporting**: dev-vs-held-out gap computed and reported
  (0 pts for both).
- **Cost-as-signal**: every figure read from provider JSON, framed against
  human-labor value, never used as a gate.
- **Findings publication**: this log + the scorecard + the FINDINGS spectrum
  map and 002 page.

What was thin/wasteful, stated plainly:

- **The benchmark was too easy to separate the two bets.** Both hit 100% on
  both worlds. A1's gate fired 0 times across all 24 trials; A2's checker
  bounced exactly once. A tournament where neither architecture is stressed
  cannot rank them on correctness, which is exactly what happened (see
  secondary verdict).
- **Coverage mismatch**: held-out has 5 tasks, dev has 6; held-out has no
  analogue of D6 (judgment/policy refusal). "100% vs 100%" compares slightly
  different coverage, not an identical benchmark. A fuller run mirrors the task
  kinds 1:1 across worlds.
- **2 trials is too few** to call variance. 001 used 3.
- **A2's checker cost is real overhead** (~$0.84 dev / ~$0.63 held-out, roughly
  doubling enforcement spend vs A1's free gate) bought one recovered dev trial
  here. At this difficulty that is poor value, but it is not a verdict (the
  benchmark did not stress either).

The machinery is sound; the benchmark difficulty was the limiting factor, not
the rig. That is the cheap lesson this lean cut was designed to surface.

### Secondary result: A1 vs A2 is INCONCLUSIVE.

Neither architecture clearly wins. Both pass dev and held-out at 100% with zero
safety-floor failures and a 0-point generalization gap. On correctness +
safety + generalization (the ranking axes), they are tied. The benchmark was
not hard enough to separate them:

- A1's code-gate **never fired** (0/24). 001's known weakness (topic-blind
  over-firing on escalation phrasing) did not surface here, so we did not even
  observe the precision cost that would count against A1.
- A2's checker **fired once** (dev D3 trial-2): the doer asked a clarifying
  question instead of writing the approval artifact; the checker caught the
  missing artifact, bounced one correction, and the doer then filed an on-topic
  approval. That trial would have failed `approval_file_written` without the
  bounce, so the checker recovered a would-be dev miss A1's task happened not to
  hit (A1 passed D3 both trials). One recovered trial on the dev (visible) world
  is a signal, not a generalization result: it did not recur on held-out.

Cost is the only separating signal, and per PROCESS.md cost is a
tiebreaker/signal, not a disqualifier. A2 costs ~$1.05 more enforcement spend on
dev for one recovered trial at this difficulty. Framed against human-labor
value, both are cents-to-a-third-of-a-dollar per task, far below the assistant
labor they stand in for; cost is not binding at this maturity. So cost favors
A1 marginally, but on a benchmark that stressed neither, that is not enough to
declare a winner.

**A divergent bet that does not get stressed is not a failure of the bet; it is
a failure of the benchmark to discriminate.** A2 did not break (zero safety
failures, recovered one trial). A1 did not break (zero failures, zero waste).
We cannot rank them on this evidence.

### What a fuller run would settle.

The single experiment that resolves H-16 and the A1-vs-A2 question:

- **Harder, contract-stressing tasks** where the doer is *tempted* to defer in
  chat or hand-edit, including escalation phrased ambiguously (to make A1's
  vocabulary-keyed gate over-fire and reveal its precision cost) and on-topic vs
  off-topic escalation traps (to test whether A2's checker's topic-awareness is
  actually load-bearing vs A1's topic-blind gate).
- **1:1 task-kind coverage** across dev and held-out (give held-out a judgment
  analogue of D6).
- **3 trials** minimum, and at least a second held-out world.
- **An A3 null** (no gate, constrained prose only) to re-confirm 001's leg-1
  finding at this larger brain, since 002 never ran prose-only here.

Until then: H-16 stays SUPPORTED-but-thin, and the A1-vs-A2 ranking is open.

### Decision rights.

No safety-floor failure occurred and no budget/iteration cap was reached, so no
operator stop was triggered. **Scaling 002 up (more worlds, more and harder
tasks, more architectures, an A3 null) is an operator-level decision** per the
charter and PROCESS.md decision-rights table: it is an architecture-level
direction change beyond the lean charter's scope.
